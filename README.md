# AX Marketing Partners Bot

Telegram-бот для масової розсилки повідомлень у партнерські чати.
Підтримує текст, фото, відео, документи, аудіо, голосові та кружки (video note).

---

## Як це працює

```
Управляючий чат (ти)
        │
        │  /broadcast → надсилаєш контент → підтверджуєш
        ▼
      БОТ
        │
        ├──► Чат партнера 1
        ├──► Чат партнера 2
        ├──► ...
        └──► Чат партнера N
```

Бот мовчить в партнерських чатах — тільки приймає і розсилає.

## Вимоги

- Docker + Docker Compose

## Швидкий старт

### 1. Отримати токен бота

1. Напиши [@BotFather](https://t.me/BotFather) → `/newbot`
2. Скопіюй токен виду `123456789:ABCdef...`

### 2. Підготувати управляючий чат

1. Створи закриту групу «Розсилки партнерам»
2. Додай туди себе та бота
3. Дізнайся ID цієї групи — надішли в неї будь-яке повідомлення, потім відкрий:
   ```
   https://api.telegram.org/bot<TOKEN>/getUpdates
   ```
   Знайди `"chat":{"id":` — це і є `ADMIN_CHAT_ID` (від'ємне число, наприклад `-1001234567890`)

   Альтернатива: додай [@username_to_id_bot](https://t.me/username_to_id_bot) в групу.

### 3. Налаштувати .env

```bash
cp .env.example .env
```

Заповни два поля:

```env
BOT_TOKEN=123456789:ABCdef...
ADMIN_CHAT_ID=-1001234567890
```

### 4. Запустити

```bash
docker compose up -d
```

Перевір логи:

```bash
docker compose logs -f
```

Якщо бачиш `[Bot] @your_bot started` — все працює.

## Додавання партнерських чатів

1. Додай бота адміністратором у партнерський чат
2. Бот автоматично збереже чат і надішле сповіщення в управляючий чат:
   ```
   ✅ Доданий до: Назва чату
   ID: -100987654321
   ```

Якщо бота видалять з чату — він автоматично прибере його зі списку.

## Команди

Всі команди працюють **тільки в управляючому чаті**.

| Команда | Що робить |
|---------|-----------|
| `/start` | Статус бота: кількість чатів, остання розсилка |
| `/broadcast` | Запустити розсилку |
| `/list` | Список підключених чатів із датою останньої розсилки |
| `/checkchats` | Перевірити чи бот досі активний у кожному чаті |
| `/removechat` | Видалити чат зі списку (з пагінацією) |
| `/cancel` | Скасувати поточну дію або заплановану розсилку |
| `/help` | Інструкція |

## Розсилка — покроково

```
/broadcast
  └─ надішли повідомлення (будь-який тип медіа)
       └─ вибери дію:
            ✅ Всім (N)         — відправити зараз у всі чати
            🎯 Вибрати чати    — обрати окремі чати через checkbox
            ⏰ 30 хв / 1 год   — відкласти відправку
            ⏰ 2 год / Завтра  — відкласти відправку
            ❌ Скасувати
```

Після розсилки — звіт:
```
📊 Звіт розсилки:
✅ Успішно: 48
❌ Помилок: 2

Деталі:
• Чат партнера А: bot was kicked
• Чат партнера Б: chat not found

🗑 Видалити недоступні (2)
```

---

## Перевірка чатів

`/checkchats` перевіряє статус бота без відправки повідомлень:

```
Статус чатів (50):

✅ Чат партнера А
✅ Чат партнера Б
⚠️ Чат партнера В  (учасник, не адмін — можливо немає прав на відправку)
❌ Чат партнера Г

🗑 Видалити недоступні (1)
```

Запускай раз на тиждень щоб підтримувати список в порядку.

## Деплой на сервер

```bash
# Скопіюй проект на сервер
scp -r . user@your-server:/opt/ax-bot

# На сервері
cd /opt/ax-bot
cp .env.example .env
nano .env   # ← встав BOT_TOKEN і ADMIN_CHAT_ID

# Запуск
docker compose up -d
```

### Корисні команди

```bash
# Логи в реальному часі
docker compose logs -f

# Перезапустити (після змін .env)
docker compose restart

# Оновити бота після змін у коді
docker compose up -d --build

# Зупинити
docker compose down
```

### Дані

Список чатів зберігається у `./data/chats.json` на хості.
При оновленні або перезапуску бота — дані зберігаються.

> ⚠️ **Заплановані розсилки** скасовуються при перезапуску контейнера.
> Після `docker compose restart` заплановану розсилку треба налаштувати заново.
