# AX Marketing Partners Bot

Telegram-бот для масової розсилки повідомлень у партнерські чати.
Підтримує текст, фото, відео, документи, аудіо, голосові та кружки (video note).

---

## Як це працює

```
Управляючий чат (ти)
        │
        │  /broadcast → надсилаєш контент → підтверджуєш
        ▼
      БОТ
        │
        ├──► Чат партнера 1
        ├──► Чат партнера 2
        ├──► ...
        └──► Чат партнера N
```

Бот мовчить в партнерських чатах — тільки приймає і розсилає.

---

## Вимоги

- Docker + Docker Compose
- Telegram-бот (токен від [@BotFather](https://t.me/BotFather))
- VPS або будь-який сервер з Docker

## Швидкий старт

### 1. Отримати токен бота

1. Напиши [@BotFather](https://t.me/BotFather) → `/newbot`
2. Скопіюй токен виду `123456789:ABCdef...`

### 2. Підготувати управляючий чат

1. Створи закриту групу «Розсилки партнерам»
2. Додай туди себе та бота
3. Дізнайся ID цієї групи — надішли в неї будь-яке повідомлення, потім відкрий:
   ```
   https://api.telegram.org/bot<TOKEN>/getUpdates
   ```
   Знайди `"chat":{"id":` — це і є `ADMIN_CHAT_ID` (від'ємне число, наприклад `-1001234567890`)

   Альтернатива: додай [@username_to_id_bot](https://t.me/username_to_id_bot) в групу.

### 3. Налаштувати .env

```bash
cp .env.example .env
```

Заповни два поля:

```env
BOT_TOKEN=123456789:ABCdef...
ADMIN_CHAT_ID=-1001234567890
```

### 4. Запустити

```bash
docker compose up -d
```

Перевір логи:

```bash
docker compose logs -f
```

Якщо бачиш `[Bot] @your_bot started` — все працює.

## Додавання партнерських чатів

**Швидкий спосіб** — команда `/addlink` в управляючому чаті:

```
/addlink
```

Бот надішле два посилання:
- **Для групи** — клік → вибираєш групу → Telegram пропонує права адміна → підтверджуєш
- **Для каналу** — те саме

Після підтвердження бот з'явиться в чаті як адмін і одразу надішле в управляючий чат:
```
✅ Доданий до: Назва чату
ID: -100987654321
```

**Вручну:** додай бота в чат → зроби адміністратором → бот збережеться автоматично.

Якщо бота видалять з чату — він автоматично прибере його зі списку.

---

## Команди

Всі команди працюють **тільки в управляючому чаті**.

| Команда | Що робить |
|---------|-----------|
| `/start` | Статус бота: кількість чатів, остання розсилка, кількість запланованих |
| `/broadcast` | Запустити розсилку |
| `/scheduled` | Переглянути та скасувати заплановані розсилки |
| `/addlink` | Посилання для швидкого додавання бота в групу або канал |
| `/addchat` | Додати чат вручну за ID або пересланим постом групи/каналу |
| `/removedchats` | Переглянути та відновити видалені чати |
| `/list` | Список підключених чатів із датою останньої розсилки |
| `/checkchats` | Перевірити чи бот досі активний у кожному чаті |
| `/removechat` | Видалити чат зі списку (з пагінацією) |
| `/cancel` | Скасувати поточну активну дію |
| `/help` | Інструкція |

## Розсилка — покроково

```
/broadcast
  └─ надішли повідомлення (будь-який тип медіа)
       └─ вибери дію:
            ✅ Всім (N)         — відправити зараз у всі чати
            🎯 Вибрати чати    — обрати окремі чати через checkbox
            ⏰ 30 хв / 1 год   — відкласти відправку
            ⏰ 2 год           — відкласти відправку
            🕐 Свій час        — ввести точний HH:MM для відправки
            ❌ Скасувати
```

> Можна мати **кілька запланованих розсилок одночасно** — кожна отримує свій токен і відображається в `/scheduled`.

Після розсилки — звіт:
```
📊 Звіт розсилки:
✅ Успішно: 48
❌ Помилок: 2

Деталі:
• Чат партнера А: bot was kicked
• Чат партнера Б: chat not found

🗑 Видалити недоступні (2)
```

## Заплановані розсилки

`/scheduled` показує всі заплановані розсилки з кнопкою скасування для кожної:

```
Заплановані розсилки (2):

1. ⏰ 18:30 (сьогодні)
2. ⏰ 10:00 (завтра)

[❌ Скасувати 18:30 (сьогодні)]
[❌ Скасувати 10:00 (завтра)]
```

> ⚠️ Заплановані розсилки зберігаються тільки **в пам'яті**. При перезапуску контейнера вони скасовуються.

---

## Відновлення видалених чатів

Коли бота видаляють з чату — чат **не знищується**, а лише помічається як видалений (soft-delete).

`/removedchats` показує такі чати з кнопкою для відновлення:

```
Видалені чати (2):

1. Чат партнера А  (видалено 3 год тому)
2. Чат партнера Б  (видалено вчора)

[🔄 Чат партнера А]
[🔄 Чат партнера Б]
```

Після відновлення чат повертається до списку розсилки. Якщо бота знову додають в чат — він відновлюється автоматично.

---

## Перевірка чатів

`/checkchats` перевіряє статус бота без відправки повідомлень:

```
Статус чатів (50):

✅ Чат партнера А
✅ Чат партнера Б
⚠️ Чат партнера В  (учасник, не адмін — можливо немає прав на відправку)
❌ Чат партнера Г

🗑 Видалити недоступні (1)
```

Запускай раз на тиждень щоб підтримувати список в порядку.

## Деплой на сервер

```bash
# Скопіюй проект на сервер
scp -r . user@your-server:/opt/ax-bot

# На сервері
cd /opt/ax-bot
cp .env.example .env
nano .env   # ← встав BOT_TOKEN і ADMIN_CHAT_ID

# Запуск
docker compose up -d
```

### Корисні команди

```bash
# Логи в реальному часі
docker compose logs -f

# Перезапустити (після змін .env)
docker compose restart

# Оновити бота після змін у коді
docker compose up -d --build

# Зупинити
docker compose down
```

### Дані

Список чатів зберігається у `./data/chats.json` на хості.
При оновленні або перезапуску бота — дані зберігаються.

> ⚠️ **Заплановані розсилки** скасовуються при перезапуску контейнера.
> Після `docker compose restart` заплановану розсилку треба налаштувати заново.
